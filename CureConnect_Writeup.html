1. Abstract 1 Mark

CureConnect is a web-based healthcare platform that integrates the MyUpchaar API to deliver real-time medicine search, prescription management, and pharmacy connectivity. Built using React, Firebase, and Vite, the system follows a structured data engineering pipeline — from API ingestion to cloud storage and user analytics via PostHog. The platform bridges the gap between patients and healthcare partners through a unified, scalable solution.

2. Introduction 2 Marks

Healthcare services today are scattered and inefficient. Patients must visit multiple platforms to search for medicines, upload prescriptions, and contact pharmacies — leading to delays and a poor user experience.

CureConnect addresses this by providing a single unified platform. Its key innovation is the integration of the MyUpchaar API, which acts as the real-time data engine supplying live medicine names, prices, and availability.

The platform consists of four core modules:

  • Medicine Search — powered by MyUpchaar API via myUpcharService.ts
  • Prescription Upload — stored securely using Firebase Storage
  • Partner Login — pharmacies and clinics manage orders via a dedicated portal
  • Analytics — PostHog tracks user interactions for continuous improvement

The objective is to demonstrate how modern data engineering principles — API integration, cloud storage, real-time data flow, and behavioral analytics — can be applied to solve a real-world healthcare problem.

3. Methodology 5 Marks

3.1 Architecture (2.5 Marks)

CureConnect follows a 4-layer data engineering architecture:

LAYER 1 — DATA SOURCE  |  MyUpchaar API → Live medicine data
↓
LAYER 2 — SERVICE LAYER  |  myUpcharService.ts → API calls & response parsing
↓
LAYER 3 — BACKEND  |  Firebase Auth + Firestore + Storage
↓
LAYER 4 — FRONTEND + ANALYTICS  |  React + Vite UI  |  PostHog Tracking

Data Flow:

User enters query → myUpcharService.ts calls MyUpchaar API → API returns medicine name, price, availability → React UI renders the results → User adds to cart → Firebase Firestore saves the order → Partner (pharmacy) receives notification via Firestore trigger → PostHog logs the complete interaction

3.2 Pseudo Code (2.5 Marks)

A) Medicine Search using MyUpchaar API:

START User enters medicine name in search bar CALL myUpcharService.searchMedicine(query) → Send GET request to MyUpchaar API → Receive response: {name, price, availability} IF response is successful: Display medicine list on screen IF user clicks "Add to Cart": Save order details to Firebase Firestore Notify pharmacy partner via Firestore trigger Log event "medicine_searched" to PostHog END

B) Prescription Upload:

START User logs in via Firebase Auth User selects prescription image file Upload image to Firebase Storage Save metadata to Firestore: { userID, fileName, timestamp, status: "pending" } Partner logs in and views pending prescriptions Partner updates status → "approved" / "rejected" User sees real-time status update on dashboard END

4. Conclusion 2 Marks

CureConnect successfully applies data engineering concepts to a real-world healthcare scenario. The MyUpchaar API serves as the primary data source, demonstrating live API ingestion and processing. Firebase provides a reliable cloud backend for storage and authentication, while PostHog enables data-driven decision making through behavioral analytics.

The platform is fully functional and deployment-ready on Firebase Hosting. Future enhancements include:

  • AI-based prescription text recognition (OCR)
  • Offline caching of MyUpchaar API responses using Firestore
  • Mobile application using the same service layer

CureConnect proves that combining a third-party healthcare API (MyUpchaar) with modern cloud infrastructure can create scalable, impactful solutions for the healthcare domain.
